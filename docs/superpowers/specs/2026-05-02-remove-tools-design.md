# Conan 工具功能移除设计规范

## 概述

本规范描述了从 Conan 数字平台中完全移除工具功能的技术方案。根据用户要求，我们将：
1. 完全移除所有工具功能代码和相关依赖
2. 保留 `/app/tools` 路由，显示一个简单的占位符页面
3. 清理未使用的代码和依赖项，保持代码库整洁

## 设计决策

### 选择的方法：完全清理（方法二）
- **理由**：用户明确要求完全移除工具相关代码和依赖
- **优势**：最彻底的清理，减少代码维护负担
- **风险**：一次性更改较多，需要仔细验证构建

### 路由策略：保留路由但显示空状态
- 保持现有路由配置不变，避免影响导航结构
- 显示用户友好的占位符信息，而非完全移除路由
- 为未来可能的重新设计保留路由入口

### 依赖清理：完全移除相关依赖
- 移除工具专用依赖 `crypto-js`
- 保持其他通用依赖不变
- 更新 lock 文件确保依赖一致性

## 技术规范

### 1. 文件变更清单

#### 1.1 需要修改的文件
| 文件路径 | 操作 | 详细说明 |
|---------|------|----------|
| `packages/web/src/views/Tools.vue` | 完全重写 | 替换为简单的占位符页面，移除所有工具逻辑（1089行代码缩减到约20行） |
| `packages/web/src/components/tools/JsonFormatter.vue` | 删除 | 工具组件目录中的唯一文件 |
| `packages/web/src/components/tools/` | 删除目录 | 移除整个工具组件目录 |
| `packages/web/package.json` | 修改 | 移除 `crypto-js: "^4.2.0"` 依赖项 |
| `pnpm-lock.yaml` | 自动更新 | 通过 `pnpm install` 命令自动更新 |

### 2. 依赖清理规范

#### 2.1 需要移除的依赖
```json
"crypto-js": "^4.2.0"
```

#### 2.2 清理流程
1. 从 `packages/web/package.json` 的 `dependencies` 中移除 `crypto-js`
2. 运行 `pnpm install` 更新 `pnpm-lock.yaml`
3. 验证 `node_modules/.pnpm/crypto-js@*` 目录已移除

#### 2.3 其他依赖评估
- `@types/crypto-js`：开发依赖，随主依赖一同移除
- 其他通用依赖（axios、vue、pinia等）保持不变

### 3. 构建和验证规范

#### 3.1 构建验证
1. **开发构建**：`pnpm --filter web dev` 应正常启动
2. **生产构建**：`pnpm build` 应成功完成
3. **类型检查**：`vue-tsc -b` 应通过类型检查

#### 3.2 功能验证
1. **路由访问**：`/app/tools` 应显示占位符页面
2. **导航测试**：侧边栏导航到工具页面应正常工作
3. **认证检查**：登录状态下应能正常访问工具页面
4. **其他功能**：确保笔记、统计等其他功能未受影响

#### 3.3 代码质量检查
1. **未使用代码检查**：确认无其他工具相关代码残留
2. **导入检查**：验证无对 `crypto-js` 的残留导入
3. **构建产物检查**：确保构建产物不包含工具相关代码

## 风险与缓解

### 风险 1：构建失败
- **可能性**：中
- **影响**：开发和生产构建失败
- **缓解措施**：
  - 逐步实施，先重写页面再移除依赖
  - 保留备份文件以便快速回滚
  - 仔细检查类型错误和导入问题

### 风险 2：路由配置问题
- **可能性**：低
- **影响**：工具页面无法访问或404错误
- **缓解措施**：
  - 保持路由配置完全不变
  - 验证路由守卫逻辑不受影响
  - 测试登录状态下的页面访问

### 风险 3：残留依赖
- **可能性**：低
- **影响**：未使用的依赖增加包体积
- **缓解措施**：
  - 运行 `pnpm ls crypto-js` 验证完全移除
  - 检查构建产物大小变化

### 风险 4：用户界面不协调
- **可能性**：低
- **影响**：占位符页面与整体设计不协调
- **缓解措施**：
  - 使用现有 UI 组件和样式类
  - 保持与整体设计语言一致
